La reproducibilidad es un requisito esencial del trabajo científico. Este capítulo
documenta los elementos que permiten reproducir los resultados presentados: el
repositorio de código, el entorno de ejecución, la contenedorización, el control de la
aleatoriedad, la trazabilidad de los experimentos y la integración continua.

\section{Repositorio y estructura del proyecto}
\label{sec::repro_repo}

El código fuente completo está disponible en un repositorio público de GitHub
% [PENDIENTE: insertar la URL exacta del repositorio del proyecto, p. ej.
%  \url{https://github.com/SkullSupernova/MDL_TFM}, y verificarla.]
y se organiza en módulos diferenciados: el código fuente (\texttt{src/}), la
configuración (\texttt{config/}), las pruebas (\texttt{test/}), la documentación
(\texttt{docs/}) y la definición de la integración continua (\texttt{.github/}). Los
artefactos pesados —conjunto de datos, modelos entrenados y resultados de
experimentos— quedan fuera del control de versiones.

\section{Entorno de ejecución}
\label{sec::repro_entorno}

El proyecto se desarrolla en \textbf{Python 3.12}. Las dependencias se fijan en
\texttt{requirements.txt}. El entrenamiento se realizó con PyTorch sobre GPU
(NVIDIA RTX 4060, CUDA), mientras que la inferencia en producción se ejecuta en CPU
para maximizar la portabilidad.

\section{Contenedorización}
\label{sec::repro_docker}

Para eliminar la dependencia del entorno local, el sistema se distribuye como imagen
Docker. Existen dos vías de despliegue: la construcción desde el código fuente
(Anexo~\ref{anexo::docker-local}) y la descarga de la imagen prepublicada en GitHub
Container Registry (Anexo~\ref{anexo::docker-ghcr}).

\section{Control de la aleatoriedad}
\label{sec::repro_semilla}

Todos los experimentos emplean una \textbf{semilla fija (42)} que controla la
inicialización de los pesos, el orden de los datos y demás fuentes de aleatoriedad.

\section{Trazabilidad de los experimentos}
\label{sec::repro_trazabilidad}

Cada ejecución genera una carpeta autocontenida (\texttt{experiments/<run\_id>/}) con
la configuración, las métricas, las curvas, las predicciones y un informe. Un fichero
\texttt{manifest.json} registra el \textit{hash} de Git, el entorno, el hardware y la
suma de verificación SHA-256 del \textit{checkpoint}, lo que vincula cada resultado con
el estado exacto del código que lo produjo. El índice \texttt{experiments/leaderboard.csv}
agrega todas las ejecuciones, y \texttt{leaderboard\_ci.csv} sus intervalos de confianza.

\section{Integración continua y pruebas}
\label{sec::repro_ci}

El repositorio incorpora una batería de pruebas automatizadas (\textit{pytest}) y un
flujo de integración continua en GitHub Actions que las ejecuta en cada cambio,
garantizando que el código permanece funcional.
